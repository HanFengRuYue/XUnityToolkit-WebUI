# XUnityToolkit-WebUI 教程视频文案

【画面1：一个 Unity 游戏的英文/日文界面，玩家正在操作但看不懂文字】

你是否遇到打算玩一个游戏游戏，但是界面全是英文或者日文，完全看不懂在讲什么。怎么办呢？去搜汉化补丁，要么根本没有，要么版本对不上。自己装翻译插件？GitHub 在国内又经常打不开，好不容易打开了，BepInEx和XUnity分好几个版本不知道下哪个，手动部署文件放错目录游戏就起不来。就算折腾好了，插件自带的谷歌翻译、百度翻译的机器翻译翻出来的东西也是一言难尽，角色名乱翻，语句不通顺。翻译完了还可能碰到方块字，一堆"口口口"看都看不了。



【画面2：切换到 XUnityToolkit 的启动画面和游戏库主界面】

现在最新推出的XUnity工具箱就是来解决这些问题的。一键安装翻译框架，不用手动去 GitHub 下载部署；翻译可以使用 AI 大模型，质量比传统机翻好得多；字体问题也能一键修复。今天我就从头到尾演示一遍怎么用。



【画面：显示下载页面，三个版本并排展示】

此工具箱完全MIT开源免费，你可以到项目的GitHub仓库进行下载，也可以到视频下方简介的网盘链接中进行下载。如分享的文件中显示，第一个是安装包版本，第二个是压缩包，解压直接运行的版本。现在我演示的是下载最新的安装包。



【画面：解压/安装完成，双击启动，系统托盘显示图标，WebView2 窗口打开】

装好之后双击启动。第一次打开你会看到一个空的游戏库，现在我们来添加第一个游戏。

【画面：点击"添加游戏"按钮，弹出文件夹选择器】

点这个"添加游戏"，选到你游戏的安装目录就行。

【画面：选择完文件夹后，工具自动检测，游戏卡片出现在游戏库中，显示封面和游戏信息】

选完之后工具会自动帮你检测。它会识别这个游戏是 Mono 还是 IL2CPP 引擎，是 32 位还是 64 位。如果你的游戏目录里有 Steam 的信息文件，它还会自动拉取游戏名称和封面图。

【画面：鼠标悬停在游戏卡片上，展示状态标识"未安装"】

你看，游戏已经加进来了，但是状态是"未安装"，说明翻译框架还没有部署上去。

【画面：补充演示——用"自动检测添加"方式，选择一个包含多个游戏的文件夹，批量检测结果】

顺便说一下，如果你有很多游戏想一起加，可以用"自动检测添加"。选一个大的游戏目录，比如你的 Steam 游戏库文件夹，工具会自动扫描里面所有的 Unity 游戏，批量添加进来，省得一个一个选。

【画面：点击游戏卡片进入游戏详情页，展示游戏详情布局——背景图、游戏名、安装按钮】

点进这个游戏的详情页。这里就是你对这个游戏做所有操作的地方。现在最重要的一步——安装翻译框架。

【画面：点击"安装"按钮，安装进度面板展开，显示各步骤实时状态】

点"安装"就行了。你能看到安装进度，它会依次完成下载框架文件、部署 BepInEx、部署 XUnity.AutoTranslator、配置翻译端点这些步骤。整个过程全自动，你不用管任何事情。

【画面：安装完成，状态变为绿色"已安装"】

好，安装完成了。状态变成了绿色，说明翻译框架已经全部就位。

【画面：鼠标指向安装状态旁边的"卸载"按钮】

万一装完之后游戏出了什么问题，别担心，这里有个"卸载"按钮。工具在安装的时候已经帮你做好了备份，卸载会把所有文件完整还原，不会留下任何残余。

【画面：切换到设置页面，展示 AI 翻译配置区域】

框架装好了，接下来要告诉工具用哪个 AI 来翻译。点开设置页面，找到 AI 翻译这一块。

【画面：展示提供商下拉列表，显示 OpenAI、Claude、DeepSeek 等选项】

这里支持的 AI 服务挺多的：OpenAI、Claude、Gemini、DeepSeek、通义千问、智谱 GLM、Kimi，还有任何兼容 OpenAI API 格式的自定义服务都能接。

【画面：选择 DeepSeek，填入 API Key，选择模型】

我这里拿 DeepSeek 举例。选择提供商，填上你的 API Key。密钥会用 Windows DPAPI 加密存储在本地，不用担心泄露的问题。模型可以手动填，也可以点"获取模型列表"让它自动拉取你账号下可用的模型。

【画面：点击"测试"按钮，显示测试成功】

填完之后点"测试"验证一下，成功了就说明配置没问题。

【画面：添加第二个提供商，展示负载均衡的说明】

这里有个小技巧，你可以同时配置好几个提供商。比如同时加上 DeepSeek 和通义千问，系统会自动把翻译请求分散到这些提供商上，翻译速度更快，而且其中一个挂了也不影响。

【画面：回到游戏详情页，点击"启动游戏"按钮】

好，配置完了。现在直接启动游戏。

【画面：游戏运行中，界面文字从英文/日文变成中文，实时翻译效果】

你看，游戏里的文字已经开始自动翻译了。XUnity.AutoTranslator 会捕获界面上出现的文字，发给我们配置好的 AI 去翻译，翻译完之后实时替换到游戏界面上。已经翻译过的文字会被缓存起来，下次碰到同样的内容就直接用缓存结果，不用重复翻译。

【画面：切换到 AI 翻译页面，展示实时翻译统计——管线分叉图、翻译数、成功率、队列】

切到"AI 翻译"页面，你能看到实时的翻译统计。这里有个管线图，展示了翻译请求的流向：接收进来的文本，走 LLM 翻译的有多少条在排队、翻译中、已完成，走翻译记忆命中的又有多少。下面还有成功率、最近的翻译记录和错误信息，方便你随时掌握翻译的运行情况。

【画面：游戏中出现方块字"口口口"的画面】

等等，有些同学可能遇到过一个问题：翻译是翻译了，但游戏里显示的全是方块——"口口口口口"。这是什么情况？

【画面：一段简单的解释动画或文字说明——字体不包含中文字符】

这是因为游戏用的 TextMesh Pro 字体本身不包含中文字符。你把文字翻译成中文了，但字体里找不到对应的字，就只能显示成方块。解决方法就是把游戏的字体替换成支持中文的。

【画面：进入游戏详情页，点击"字体替换"】

回到游戏详情页，点"字体替换"。

【画面：点击"扫描"按钮，显示扫描结果——列出游戏中所有 TMP 字体】

先点"扫描"，工具会自动扫描游戏文件里所有的 TMP 字体。

【画面：扫描结果列表，显示字体名称等信息】

扫描出来了，你可以看到游戏用了哪些字体。

【画面：选择要替换的字体，选择内置 TMP 字体，点击"替换"】

选上你要替换的字体，然后选择替换源。最简单的方式就是用工具内置的中文 TMP 字体，它会自动匹配你游戏的 Unity 版本。点"替换"就搞定了。

【画面：替换完成后重新启动游戏，中文正常显示】

再打开游戏看看，中文字体正常显示了，再也没有方块字了。

【画面：展示"还原"按钮】

放心，替换之前工具已经帮你备份了原始字体文件。如果你想恢复，随时点"还原"就行。

【画面：展示字体生成器页面入口】

如果你不满意内置字体的效果，想用自己喜欢的字体，可以用"字体生成器"。上传一个 TTF 或 OTF 中文字体文件，选好字符集——GB2312 覆盖日常用字，GBK 更全一些，CJK 就是最完整的覆盖——然后点生成，等它处理完就能下载或者直接安装到游戏里。

【画面：切换到一个文字特别多的 RPG/视觉小说游戏画面，文字加载时有明显的等待】

好了，基础的功能咱们都会用了。接下来聊一些进阶的东西。

如果你玩的是文字量特别大的游戏，比如 RPG 或者视觉小说，每次碰到新对话都要等 AI 翻译，体验就不太好了。有没有办法提前把文字全部翻译好，进游戏就是中文？

【画面：进入游戏详情页，点击"资产提取"】

有的，这就是"预翻译"功能。先到游戏详情页点"资产提取"。

【画面：点击"提取"按钮，显示提取进度和结果——提取到的文本数量】

点"提取"，工具会直接扫描游戏的资源文件，把里面所有的文本字符串都提取出来。

【画面：展示提取结果，大量文本条目】

你看，提取出来了这么多条文本。这些就是游戏里所有可翻译的文字。

【画面：点击"预翻译"按钮，显示预翻译进度——进度条、当前阶段、翻译数量】

现在点"预翻译"，工具会把这些文本批量发给 AI 翻译。你能看到实时的进度，包括当前翻译到了哪个阶段、已经完成了多少条。

【画面：展示暂停/取消按钮】

预翻译可以随时暂停或者取消，不用担心。已经翻译好的部分会保存下来，下次继续的时候自动跳过已完成的条目，不会重复翻译。

【画面：预翻译完成后启动游戏，文本加载速度极快，进入对话界面直接显示中文】

预翻译完成之后再启动游戏，所有文字直接从缓存加载，秒出中文，不用等了。

【画面：游戏中某个角色名或地名被翻译得不一致——同一个名字在不同地方翻译成了不同的中文】

玩翻译游戏的时候你可能还会碰到一个问题：同一个角色的名字，AI 一会儿翻成这样一会儿翻成那样，或者有些专有名词你希望它不要翻译，保留原文。

【画面：进入游戏详情页，点击"术语编辑器"】

这种时候就要用到"术语编辑器"了。进入游戏详情页，点"术语编辑器"。

【画面：添加翻译术语的操作——输入原文 "Dragonborn"，译文 "龙裔"，类型选择"翻译"】

比如你希望 "Dragonborn" 始终翻译成"龙裔"，就添加一条翻译术语，原文填 Dragonborn，译文填龙裔。

【画面：添加禁翻术语的操作——输入 "Fireball"，类型选择"禁翻"】

再比如有些技能名你想保留英文原名，比如 Fireball，那就添加一条禁翻术语，AI 在翻译的时候就会保留这个词不动。

【画面：展示术语属性配置——分类、优先级、大小写敏感等选项】

每条术语还可以设置分类、优先级这些属性，让管理更方便。

【画面：展示 AI 描述编辑区域】

还有一个很有用的功能——"AI 描述"。你可以在这里写上这个游戏的世界观、背景设定，帮助 AI 更好地理解游戏的上下文。比如告诉它这是一个中世纪奇幻游戏，主角是一名骑士。这样 AI 翻译的时候就会更贴合游戏的语境。

【画面：展示多阶段翻译管线的示意图——自然翻译→占位符替换→强制修正】

工具内部其实有一套多阶段的翻译管线来保证术语的准确性。第一阶段先把术语以提示词的方式告诉 AI，让它自然地翻译。如果 AI 没有按要求来，第二阶段会用占位符强制替换。还不行的话，第三阶段会做最终的强制修正。这些都是自动的，你不用操心。

【画面：展示 AI 翻译设置中的"自动提取术语"开关】

另外，你还可以开启"自动提取术语"这个功能。开启之后，AI 在翻译的过程中会自动识别出游戏里的专有名词，比如角色名、地名、物品名，然后自动添加到术语表里。你只需要检查一下看看有没有不对的，省去了自己一个个添加的麻烦。

【画面：切换到翻译编辑器页面，展示原文和译文的对照列表】

翻译完之后，如果你想对结果做一些人工校对，可以用"翻译编辑器"。

【画面：在编辑器中选择一条翻译，直接修改译文】

这里列出了所有已翻译的条目，原文和译文对照着看。直接点进去就能编辑，改完立刻生效。

【画面：展示导出/导入按钮】

翻译结果还可以导出成文件，方便你备份或者分享给别人。同样的，你也可以导入别人翻译好的文件。

【画面：展示插件包导出功能——点击"导出"，生成 ZIP 文件】

说到分享，有一个特别实用的功能。如果你辛辛苦苦翻译完了一个游戏，想把成果分享给朋友或者社区的其他玩家，可以用"插件包导出"。

【画面：导出完成，显示 ZIP 文件】

它会把翻译框架、翻译端点配置、术语表、翻译缓存这些东西全部打包成一个 ZIP 文件。

【画面：另一台电脑上，同一个游戏的详情页，点击"导入"，选择 ZIP 文件】

拿到这个 ZIP 的人，只要在同一个游戏的详情页点"导入"，一键就能还原你所有的翻译配置。不用重新设置任何东西，直接就能玩翻译好的游戏。

【画面：切换到本地大模型页面，展示 GPU 信息和模型列表】

接下来说一个很多人关心的——不想花钱买 API，或者想离线翻译怎么办？

这就是"本地大模型"的用处了。如果你下载的是 Full 完整版，工具已经内置了 llama.cpp，你只需要下载一个模型就能在本地跑 AI 翻译。

【画面：展示 GPU 信息检测结果——显卡型号、显存大小】

打开本地大模型页面，工具会自动检测你的显卡信息，告诉你显卡型号和显存大小。

【画面：展示模型目录，根据显存推荐的模型带有标识】

然后在模型目录里选择一个模型。工具会根据你的显存大小标注哪些模型适合你。显存够大就选大一点的模型，翻译质量更好；显存紧张就选小模型，速度快但质量差一些。

【画面：点击"下载"按钮，显示下载进度】

选好了点"下载"就行。下载过程中可以暂停，断了也能续传。

【画面：下载完成，调整 GPU 层数，点击"启动"】

下载完成后调整一下 GPU 层数。层数越多，模型在显卡上跑的部分就越多，速度越快，但显存占用也越大。调好之后点"启动"。

【画面：模型启动成功，点击"测试"验证】

启动成功了。点"测试"验证一下模型是否正常工作。

【画面：测试通过】

没问题。本地模型启动之后，AI 翻译会自动切换到用本地模型，你都不需要再去配置什么提供商或者 API Key。完全离线，完全免费。

【画面：展示已有 GGUF 模型文件的场景——点击"添加自有模型"】

如果你手头已经有 GGUF 格式的模型文件，不用重新下载，直接点"添加自有模型"指定文件路径就行。

【画面：切换到 BepInEx 日志分析页面】

最后再说几个实用的小功能。

万一游戏装了翻译框架之后启动不了或者闪退了，不要慌。到游戏详情页点"BepInEx 日志"。

【画面：展示日志内容，点击"AI 分析"按钮】

这里能看到 BepInEx 的运行日志。看不懂没关系，点"AI 分析"，AI 会自动帮你诊断问题原因，告诉你是什么冲突了或者哪里出了问题。

【画面：展示"卸载"按钮】

实在解决不了，还可以直接卸载翻译框架，游戏会恢复到安装前的状态。

【画面：切换到插件管理页面】

"插件管理"可以让你管理游戏里安装的 BepInEx 插件。你可以在这里启用或禁用插件，查看配置文件，甚至安装额外的插件。

【画面：切换到配置编辑页面，展示图形化配置界面和 INI 编辑器】

"配置编辑"可以让你调整 XUnity.AutoTranslator 的详细配置。有图形化的界面可以直观地修改设置，也可以直接编辑 INI 配置文件。

【画面：切换到设置页面，展示主题切换、强调色选择】

设置页面里可以切换深色和浅色主题，选择你喜欢的强调色，调整页面缩放比例。

【画面：展示检查更新功能】

工具还支持在线更新。在设置页面点"检查更新"，如果有新版本会自动下载并更新，不用手动去下载覆盖。

【画面：回到游戏库主界面，展示多个游戏的状态——有的已安装（绿色），有的未安装】

好了，以上就是 XUnityToolkit 的完整使用教程。总结一下，用这个工具你不需要手动去折腾 BepInEx 和 XUnity.AutoTranslator 的安装配置，一键搞定。不管你是想用云端 API 还是本地大模型来翻译，都有现成的方案。碰到方块字就用字体替换，想提前翻好就用预翻译，想让术语翻得准就用术语编辑器，想分享汉化成果就用插件包导出。

【画面：展示 GitHub 仓库页面】

工具是开源免费的，如果觉得好用可以去 GitHub 给个 Star。遇到问题或者有功能建议也欢迎去提 Issue。

【画面：结束画面，展示工具 Logo 和 GitHub 地址】
