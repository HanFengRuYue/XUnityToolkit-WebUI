# MSI Installer Packaging Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add WiX v5 MSI installer packaging so users get a standard Windows install/uninstall experience alongside the existing portable ZIP distribution.

**Architecture:** Per-user MSI installs to `%LocalAppData%\Programs\XUnityToolkit-WebUI\`, stores runtime data at `%AppData%\XUnityToolkit\`. WiX v5 SDK-style project (`WixToolset.Sdk`) builds MSI from publish output. Updater.exe syncs version info in HKCU uninstall registry after delta updates.

**Tech Stack:** WiX Toolset v5 (WixToolset.Sdk NuGet), PowerShell (file harvesting), Win32 Registry P/Invoke (Updater.exe AOT)

**Spec:** `docs/superpowers/specs/2026-03-15-msi-installer-design.md`

**Deviation from spec:** Spec proposed `HeatDirectory` MSBuild item for file harvesting; this plan uses custom PowerShell harvesting instead (more reliable across WiX versions, full control over output). Spec proposed `.config/dotnet-tools.json`; not needed since `WixToolset.Sdk` is a NuGet-based MSBuild SDK resolved automatically by `dotnet build`.

```xml
<Wix xmlns="http://wixtoolset.org/schemas/v4/wxs">
  <Package Name="XUnityToolkit-WebUI"
           Manufacturer="XUnityToolkit"
           Version="$(var.MsiVersion)"
           UpgradeCode="E8A3F4B2-7C1D-4E5A-9B6F-2D8C4A1E3F7B"
           Compressed="yes"
           InstallerVersion="500"
           Scope="perUser">

    <SummaryInformation Description="XUnityToolkit-WebUI Installer" />

    <MajorUpgrade AllowSameVersionUpgrades="yes"
                  DowngradeErrorMessage="已安装更新版本的 XUnityToolkit-WebUI，无法降级安装。" />

    <MediaTemplate EmbedCab="yes" CompressionLevel="high" />

    <Icon Id="AppIcon" SourceFile="$(var.PublishDir)\XUnityToolkit-WebUI.exe" />
    <Property Id="ARPPRODUCTICON" Value="AppIcon" />
    <Property Id="ARPHELPLINK" Value="https://github.com/XUnityToolkit/XUnityToolkit-WebUI" />

    <Feature Id="ProductFeature" Title="XUnityToolkit-WebUI" Level="1">
      <ComponentGroupRef Id="CoreComponents" />
      <ComponentGroupRef Id="ShortcutComponents" />
      <ComponentGroupRef Id="HarvestedFiles" />
      <ComponentGroupRef Id="CleanupComponents" />
    </Feature>

  </Package>
</Wix>
```

Key decisions:
- `UpgradeCode` is a fixed GUID (never changes across versions)
- `$(var.MsiVersion)` — WiX preprocessor variable, injected at build time via `-p:MsiVersion=x.y.z`
- `$(var.PublishDir)` — WiX preprocessor variable, injected at build time via `-p:PublishDir=...`
- `Scope="perUser"` — no UAC elevation, installs under HKCU
- `CompressionLevel="high"` — reduces MSI size for 900MB+ content

**Files:**
- Create: `Installer/Components.wxs`

- [ ] **Step 1: Create `Installer/Components.wxs`**

```xml
<?xml version="1.0" encoding="UTF-8"?>
<Wix xmlns="http://wixtoolset.org/schemas/v4/wxs">
  <Fragment>
    <ComponentGroup Id="CoreComponents" Directory="INSTALLDIR">
      <!-- Registry keys for installed mode detection -->
      <Component Id="RegistryKeys" Guid="A1B2C3D4-E5F6-7890-ABCD-EF1234567890">
        <RegistryKey Root="HKCU" Key="Software\XUnityToolkit" ForceDeleteOnUninstall="yes">
          <RegistryValue Name="DataPath"
                         Type="string"
                         Value="[AppDataFolder]XUnityToolkit"
                         KeyPath="yes" />
          <RegistryValue Name="MsiProductCode"
                         Type="string"
                         Value="[ProductCode]" />
          <RegistryValue Name="InstallDir"
                         Type="string"
                         Value="[INSTALLDIR]" />
        </RegistryKey>
      </Component>
    </ComponentGroup>
  </Fragment>
</Wix>
```

Key points:
- `DataPath` uses `Type="string"` — MSI `[AppDataFolder]` is resolved at install time to the actual path (e.g., `C:\Users\foo\AppData\Roaming\XUnityToolkit`), so REG_SZ is correct (no runtime env var expansion needed)
- `MsiProductCode` stores `[ProductCode]` for Updater.exe to locate the Uninstall registry entry
- `ForceDeleteOnUninstall="yes"` removes entire key tree on full uninstall

```xml
<?xml version="1.0" encoding="UTF-8"?>
<Wix xmlns="http://wixtoolset.org/schemas/v4/wxs"
     xmlns:util="http://wixtoolset.org/schemas/v4/wxs/util">
  <Fragment>
    <ComponentGroup Id="CleanupComponents" Directory="INSTALLDIR">
      <Component Id="CleanupAppData" Guid="D4E5F6A7-B8C9-0123-DEF0-456789ABCDEF">
        <RegistryValue Root="HKCU" Key="Software\XUnityToolkit"
                       Name="CleanupMarker" Type="integer" Value="1" KeyPath="yes" />
        <!-- RemoveFolderEx removes the folder and all contents on uninstall -->
        <util:RemoveFolderEx On="uninstall"
                             Property="APPDATAPATH" />
      </Component>
    </ComponentGroup>

    <!-- Property to hold the AppData path for RemoveFolderEx -->
    <Property Id="APPDATAPATH">
      <RegistrySearch Id="FindAppDataPath"
                      Root="HKCU"
                      Key="Software\XUnityToolkit"
                      Name="DataPath"
                      Type="directory" />
    </Property>
  </Fragment>
</Wix>
```

Key points:
- `RemoveFolderEx` from `WixToolset.Util.wixext` handles recursive directory removal correctly
- `RegistrySearch` reads the actual `DataPath` value from registry (not `%AppData%` env var)
- `On="uninstall"` — only runs during uninstall
- During MajorUpgrade, the component is re-installed by the new MSI, so the folder is preserved
- Requires a `RegistryValue` as KeyPath (WiX component requirement)

```csharp
static void SyncMsiRegistryVersion(Action<string> log)
{
    // Read MsiProductCode from our registry key
    nint hKey;
    int result = RegOpenKeyEx(
        HKEY_CURRENT_USER,
        @"Software\XUnityToolkit",
        0,
        KEY_READ,
        out hKey);

    if (result != 0)
    {
        log("  No MSI registration found (portable mode). Skipping.");
        return;
    }

    try
    {
        string? productCode = RegGetString(hKey, "MsiProductCode");
        if (string.IsNullOrEmpty(productCode))
        {
            log("  MsiProductCode not found. Skipping.");
            return;
        }

        // Read the new version from the updated executable
        string? installDir = RegGetString(hKey, "InstallDir");
        if (string.IsNullOrEmpty(installDir))
        {
            log("  InstallDir not found. Skipping.");
            return;
        }

        // Try to read InformationalVersion from the updated exe
        string exePath = Path.Combine(installDir, "XUnityToolkit-WebUI.exe");
        string? newVersion = null;
        if (File.Exists(exePath))
        {
            try
            {
                var versionInfo = System.Diagnostics.FileVersionInfo.GetVersionInfo(exePath);
                newVersion = versionInfo.ProductVersion?.Split('+')[0];
            }
            catch { /* ignore */ }
        }

        if (string.IsNullOrEmpty(newVersion))
        {
            log("  Could not determine new version. Skipping.");
            return;
        }

        // Update DisplayVersion in Uninstall registry
        string uninstallKey = @$"Software\Microsoft\Windows\CurrentVersion\Uninstall\{productCode}";
        nint hUninstall;
        result = RegOpenKeyEx(HKEY_CURRENT_USER, uninstallKey, 0, KEY_WRITE, out hUninstall);
        if (result != 0)
        {
            log($"  Could not open uninstall key: {uninstallKey}");
            return;
        }

        try
        {
            RegSetString(hUninstall, "DisplayVersion", newVersion);
            RegSetString(hUninstall, "InstallDate", DateTime.Now.ToString("yyyyMMdd"));
            log($"  Updated DisplayVersion to {newVersion}");
        }
        finally
        {
            RegCloseKey(hUninstall);
        }
    }
    finally
    {
        RegCloseKey(hKey);
    }
}

// ── Win32 Registry P/Invoke (AOT-safe) ──────────────────────────────────────

static readonly nint HKEY_CURRENT_USER = unchecked((nint)0x80000001);
const int KEY_READ = 0x20019;
const int KEY_WRITE = 0x20006;
const int REG_SZ = 1;

[System.Runtime.InteropServices.DllImport("advapi32.dll", CharSet = System.Runtime.InteropServices.CharSet.Unicode)]
static extern int RegOpenKeyEx(nint hKey, string subKey, int options, int samDesired, out nint phkResult);

[System.Runtime.InteropServices.DllImport("advapi32.dll")]
static extern int RegCloseKey(nint hKey);

[System.Runtime.InteropServices.DllImport("advapi32.dll", CharSet = System.Runtime.InteropServices.CharSet.Unicode)]
static extern int RegQueryValueEx(nint hKey, string valueName, nint reserved, out int type, char[] data, ref int dataSize);

[System.Runtime.InteropServices.DllImport("advapi32.dll", CharSet = System.Runtime.InteropServices.CharSet.Unicode)]
static extern int RegSetValueEx(nint hKey, string valueName, int reserved, int type, string data, int dataSize);

static string? RegGetString(nint hKey, string name)
{
    int type;
    int size = 520; // 260 chars * 2 bytes
    char[] buffer = new char[260];
    int result = RegQueryValueEx(hKey, name, nint.Zero, out type, buffer, ref size);
    if (result != 0 || type != REG_SZ) return null;
    return new string(buffer, 0, (size / 2) - 1); // exclude null terminator
}

static void RegSetString(nint hKey, string name, string value)
{
    int dataSize = (value.Length + 1) * 2; // include null terminator
    RegSetValueEx(hKey, name, 0, REG_SZ, value, dataSize);
}
```

Uses P/Invoke for AOT compatibility (Updater.exe has `PublishAot=true`, cannot use `Microsoft.Win32.Registry`).

**AOT note:** `System.Diagnostics.FileVersionInfo.GetVersionInfo()` uses Win32 P/Invoke internally and should be AOT-safe, but verify during testing. If it fails under AOT, fall back to reading a version file written by the main app.

```powershell
function Generate-InstallerWxs {
    param(
        [string]$ReleaseDir,
        [string]$OutputFile
    )

    Write-Host "  Generating $OutputFile from $ReleaseDir..." -ForegroundColor DarkGray

    $outputDir = Split-Path $OutputFile -Parent
    if (-not (Test-Path $outputDir)) {
        New-Item -ItemType Directory -Path $outputDir -Force | Out-Null
    }

    $xml = [System.Xml.XmlDocument]::new()
    $xml.AppendChild($xml.CreateXmlDeclaration("1.0", "UTF-8", $null)) | Out-Null

    $wixNs = "http://wixtoolset.org/schemas/v4/wxs"
    $wix = $xml.CreateElement("Wix", $wixNs)
    $xml.AppendChild($wix) | Out-Null

    $fragment = $xml.CreateElement("Fragment", $wixNs)
    $wix.AppendChild($fragment) | Out-Null

    $compGroup = $xml.CreateElement("ComponentGroup", $wixNs)
    $compGroup.SetAttribute("Id", "HarvestedFiles")
    $compGroup.SetAttribute("Directory", "INSTALLDIR")
    $fragment.AppendChild($compGroup) | Out-Null

    # Scan all files in publish output
    $files = Get-ChildItem -Path $ReleaseDir -File -Recurse
    $counter = 0

    foreach ($file in $files) {
        $relativePath = $file.FullName.Substring($ReleaseDir.TrimEnd('\').Length + 1)

        # Skip data directory if present
        if ($relativePath -like "data\*") { continue }

        $counter++
        $safeId = "f_$counter"

        # Determine directory for subdirectory files
        $dirParts = $relativePath.Split('\')

        if ($dirParts.Length -eq 1) {
            # Root-level file — add directly to component group
            $comp = $xml.CreateElement("Component", $wixNs)
            $comp.SetAttribute("Id", "c_$counter")
            $comp.SetAttribute("Guid", "*")
            $compGroup.AppendChild($comp) | Out-Null

            $fileEl = $xml.CreateElement("File", $wixNs)
            $fileEl.SetAttribute("Id", $safeId)
            $fileEl.SetAttribute("Source", "`$(var.PublishDir)\$relativePath")
            $fileEl.SetAttribute("KeyPath", "yes")
            $comp.AppendChild($fileEl) | Out-Null
        } else {
            # Subdirectory file — create directory ref + component
            $comp = $xml.CreateElement("Component", $wixNs)
            $comp.SetAttribute("Id", "c_$counter")
            $comp.SetAttribute("Guid", "*")
            $comp.SetAttribute("Subdirectory", ($dirParts[0..($dirParts.Length - 2)] -join '\'))
            $compGroup.AppendChild($comp) | Out-Null

            $fileEl = $xml.CreateElement("File", $wixNs)
            $fileEl.SetAttribute("Id", $safeId)
            $fileEl.SetAttribute("Source", "`$(var.PublishDir)\$relativePath")
            $fileEl.SetAttribute("KeyPath", "yes")
            $comp.AppendChild($fileEl) | Out-Null
        }
    }

    $xml.Save($OutputFile)
    Write-Host "  Generated $counter file entries." -ForegroundColor Green
}
```

Key points:
- Scans the publish output directory recursively
- Creates one `<Component>` per file (WiX best practice for per-file tracking)
- Uses `Guid="*"` for auto-generated GUIDs (based on file path, stable across builds)
- Uses `$(var.PublishDir)` preprocessor variable for file source paths
- `Subdirectory` attribute handles nested directories without explicit `<Directory>` nesting
- Skips `data\` directory
